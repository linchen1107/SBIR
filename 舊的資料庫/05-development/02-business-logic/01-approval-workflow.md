# 申編單送審流程與 `official_nsn` 欄位說明

## 背景
- `official_nsn` 欄位定義在 `web_app.applications`，Schema 檔位置：`sql/web_app_schema.sql`。
- 欄位用途預先預留，目前前端無對應輸入欄位。
- 該欄位代表最終核可後，由上級單位核發的正式 NATO Stock Number（NSN）。

## 送審流程摘要
1. **使用者提交申編單**  
   - 由系統記錄申請內容並送交高層審核。
2. **高層審核結果**  
   - **拒絕**：  
     - 表示資料需修正。  
     - 申請人修改內容後可再次送出，流程重新進入審核階段。  
   - **核准**：  
     - 申請內容確認無誤。  
     - 高層核發正式料號（NSN），系統應將該值寫入 `official_nsn` 欄位。
3. **資料落地**  
   - 已核准案件需保存最終 NSN，以便後續查詢、下載文件或整合外部系統。

## 待辦事項
- 前端尚未提供填寫/顯示 `official_nsn` 的介面，需要在後續核准流程整合完成後再開發。
- 需與高層審核作業介面或 API 定義資料交換格式，以便自動更新 `official_nsn`。
