# SBIR 裝備資料庫中英文對照表 (V3.2)

## 📊 資料庫資訊

- **資料庫名稱**: sbir_equipment_db_v3
- **版本**: V3.2
- **用途**: 海軍裝備管理系統 + Web 應用整合 + NSN 申編搜尋系統
- **編碼**: UTF8
- **建立日期**: 2025-11-25
- **最後更新**: 2025-12-09
- **架構特點**:
  - Item 自我關聯 BOM 結構
  - UUID 主鍵
  - 擴展表設計
  - **雙 Schema 架構**:
    - `web_app`: 裝備管理系統 + Web 應用（19 個表格，原 public schema）
    - `public`: 搜尋系統（15 個表格 + 5 個視圖）
  - 整合 Web 應用使用者與申編單管理系統
  - 統一時間戳命名（date_created/date_updated）
  - EMU3000 維修物料匯入邏輯
  - NSN 申編搜尋系統完整整合
- **總表數**: 34 個資料表（web_app: 19 + public: 15）+ 5 個視圖

## V3.2 變更說明

### 🔄 主要變更

#### 1. ✅ 新增 EMU3000 資料匯入規則

**理由**:
- 支援 EMU3000 維修物料清單的匯入
- 定義總成與次總成的識別邏輯
- 透過 BOM 結構建立關聯

**影響**:
- 新增文件章節 [EMU3000 資料匯入規則](#emu3000-資料匯入規則)
- 不影響現有資料表結構，僅定義資料對應邏輯

#### 2. ⚡ Schema 架構重構與 NSN 系統整合 (2025-12-08/09)

**理由**:
- 為了更好的資料庫組織與功能隔離
- 整合完整的 NSN 申編搜尋系統
- 支援多模組擴充架構

**第一階段：Schema 重構 (2025-12-08)**:
- 原 `public` schema 更名為 `web_app`（19 個裝備管理表）
- 建立新的 `public` schema 用於 NSN 搜尋系統
- 所有裝備管理與 Web 應用表格移至 `web_app` schema

**第二階段：NSN 系統整合 (2025-12-09)**:
- 在 `public` schema 建立 15 個 NSN 核心表格
- 建立 5 個查詢視圖（v_h6_inc_mapping 等）
- 匯入超過 50 萬筆 NSN 資料（306.8 MB）
- 完成 FSG/FSC、NATO H6、INC、FIIG、MRC 等系統整合

**影響**:
- 資料表總數：19 → 34 個
- 新增 public schema 專門用於 NSN 申編搜尋
- web_app schema 專注於裝備管理與 Web 應用
- 雙 schema 架構實現功能分離與安全隔離

## V3.1 整合變更說明

### 🔄 主要變更

#### 1. ✅ 整合 Web 應用系統（新增 6 個表）

**理由**:
- 提供完整的使用者管理系統
- 詳細的申編單管理（50+ 欄位 vs 原本 7 欄位）
- 附件管理（BYTEA 儲存）
- 完整的審計追蹤

**影響**:
- 新增 User 表（使用者管理）
- Application 表取代舊的 ApplicationForm
- 新增 ApplicationAttachment（附件管理）
- 新增 UserSession（工作階段管理）
- 新增 ApplicationLog（應用程式日誌）
- 新增 AuditLog（稽核日誌）

#### 2. ✅ 統一時間戳命名

**理由**:
- 所有表統一使用 date_created/date_updated 命名
- 保持資料庫命名一致性
- 簡化觸發器函數邏輯

**影響**:
- Web 應用表改用 date_created/date_updated
- 統一的觸發器函數
- 整個資料庫使用相同的時間戳命名規範

#### 3. ✅ Application 與 Item 表關聯

**理由**:
- 申編單需要關聯到具體的品項
- 支援申編單的完整生命週期管理

**影響**:
- Application 表新增 item_uuid 欄位
- 外鍵關聯到 Item 表

## public Schema - NSN 搜尋系統（15個表 + 5個視圖）

> **Schema**: `public`  
> **說明**: NSN 申編搜尋系統，提供 NATO 標準物品名稱查詢與申編流程  
> **建立日期**: 2025-12-08/09  
> **資料量**: 超過 50 萬筆記錄

### 系統架構

NSN 搜尋系統由以下子系統組成：

1. **FSG/FSC 分類系統**：聯邦補給群組與分類
2. **NATO H6 系統**：NATO 標準物品名稱（6 位編碼）

## EMU3000 資料匯入規則

### 來源資料
- **路徑**: `@Database/data/EMU3000/EMU 3000 維修物料清單/5年維護/01 清潔塊單元--C20、C21--OK/清潔塊單元C20、C21大修料件清單--60個月.xlsx` (範例)
- **格式**: Excel 檔案

### 識別邏輯

#### 1. 總成 (Assembly)
- **識別特徵**: `Part` 欄位為 `NULL` 或特定格式 (如 "II...")
- **主要欄位對應**:
    - `Item number` -> `Item.item_code` (識別碼)
    - `Name` -> `Item.item_name_zh` / `Item.item_name_en`
    - `Quantity` -> 不適用於 Item 表，可能用於上層 BOM
- **Item 類型**: `FG` (Finished Good) 或 `SEMI` (Semi-finished)

#### 2. 次總成/元件 (Sub-assembly / Component)
- **識別特徵**: `Part` 欄位為序號 (如 "1", "2"...)
- **主要欄位對應**:
    - `Item number` -> `Item.item_code`
    - `Name` -> `Item.item_name_zh` / `Item.item_name_en`
    - `Quantity` -> `BOM_LINE.qty_per` (單位用量)
    - `Unit` -> `BOM_LINE.uom`
- **Item 類型**: `SEMI` 或 `RM` (Raw Material)

### BOM 結構建立

1. **建立總成 Item**:
   - 將 Excel 表頭資訊或 `Part` 為 NULL 的行建立為總成 Item。
   - 建立一個對應的 `BOM` 記錄，關聯到此 Item。

2. **建立次總成 Item**:
   - 將 `Part` 為序號的每一行建立為次總成/元件 Item。

3. **建立 BOM 關聯**:
   - 在 `BOM_LINE` 表中建立記錄。
   - `bom_uuid`: 指向總成的 BOM。
   - `component_item_uuid`: 指向次總成/元件的 Item。
   - `qty_per`: 填入 Excel 中的 `Quantity`。
   - `line_no`: 填入 Excel 中的 `Part` (序號)。
